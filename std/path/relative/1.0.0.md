# std.path.relative

Returns the shortest relative path from `base` to `target`, or an error if impossible.

## Signature

(base: string, target: string) -> result[string, string]

## Behavior

Both paths are normalized. Returns a path using `../` segments to back out of `base` followed by the needed segments of `target`. Returns an error when the two are on different drives (Windows).

## Tests

+ returns the empty-ish path "." when base equals target
+ returns "c" for base "a/b" and target "a/b/c"
+ returns "../c" for base "a/b" and target "a/c"
- returns error when base and target are on different Windows drives
- does not consult the filesystem; purely syntactic

## Assumptions

? both paths treated as either both-absolute or both-relative; mixing errors
