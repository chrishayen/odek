# std.http.headers.validate_name

Returns true when a string is a valid RFC 7230 field-name (non-empty token).

## Signature

(name: string) -> bool

## Behavior

Returns true iff the string is non-empty and contains only bytes in the RFC 7230 `tchar` set: `!#$%&'*+-.^_` backtick `|~`, digits, and letters. Returns false otherwise.

## Tests

+ returns true for "Content-Type"
+ returns true for "x-custom-header"
+ returns true for a single-character token like "A"
- returns false for an empty string
- returns false when any character is outside the tchar set (space, colon, comma, etc.)
- returns false when the string contains CR or LF

## Assumptions

? token set is RFC 7230 tchar; callers using this for HTTP/2 may have slightly different rules
