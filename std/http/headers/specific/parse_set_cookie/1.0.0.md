# std.http.headers.specific.parse_set_cookie

Parses a server-side `Set-Cookie` header value into its name/value pair and attributes map.

## Signature

(value: string) -> result[set_cookie, string]

## Behavior

The first `;`-delimited segment is `name=value`; subsequent segments are attributes (`Domain`, `Path`, `Expires`, `Max-Age`, `Secure`, `HttpOnly`, `SameSite`, etc.). Attribute names are lowercased. Flag attributes (like `Secure`) have empty-string values. Returns name + value + attributes map.

## Tests

+ parses "id=abc" into name "id", value "abc", attrs {}
+ parses "id=abc; Path=/; HttpOnly" into attrs {"path": "/", "httponly": ""}
+ lowercases attribute names
- returns error when the first segment lacks "="
- returns error on an empty cookie name

## Assumptions

? attribute values are returned verbatim (no Expires parsing here)
? caller interprets Max-Age and Expires separately
