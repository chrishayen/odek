# std.http.headers.specific.parse_accept

Parses an `Accept` header into a list of (media-type, quality) pairs, preserving input order.

## Signature

(value: string) -> result[list[weighted_media_type], string]

## Behavior

Uses `parse_quality_list` to split into weighted tokens. Parses each token via `parse_content_type` into type+subtype+params. Returns a list of weighted media-types in input order. Sorting by quality is the caller's job.

## Tests

+ parses "text/html, application/json;q=0.5"
+ parses "*/*;q=0.1"
+ preserves input order
- returns error when a segment's token is not a valid media type
- returns error when q is outside [0, 1]

## Assumptions

? does not sort by q; callers that need preference order sort themselves
