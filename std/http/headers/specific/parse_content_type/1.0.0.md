# std.http.headers.specific.parse_content_type

Parses a `Content-Type` header value into its primary type, subtype, and parameters.

## Signature

(value: string) -> result[content_type, string]

## Behavior

Splits via `parse_parameters` on the base token. Splits the base at `/` into `type` and `subtype`. Lowercases both. Returns type + subtype + params.

## Tests

+ parses "text/html" into type="text", subtype="html", params={}
+ parses "application/json; charset=utf-8" into its three components
+ lowercases type and subtype
- returns error when the base lacks "/"
- returns error when type or subtype is empty
- returns error when parameters fail to parse

## Assumptions

? wildcard types ("*/*", "text/*") are accepted; caller decides meaning
