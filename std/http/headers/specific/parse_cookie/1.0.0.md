# std.http.headers.specific.parse_cookie

Parses a client-side `Cookie` header (a semicolon-separated list of `name=value` pairs) into a map.

## Signature

(value: string) -> result[map[string, string], string]

## Behavior

Splits on `; ` (semicolon + optional whitespace). For each segment, splits at the first `=` into name and value. Trims whitespace around names and values. Returns a map preserving the last-seen value for duplicate names.

## Tests

+ parses "a=1; b=2" into {"a": "1", "b": "2"}
+ handles missing whitespace: "a=1;b=2"
+ trims whitespace
+ last-wins for duplicate names
- returns error when a segment lacks "="
- returns error on an empty name

## Assumptions

? cookie name/value character rules are lenient here; strict validation is a separate rune
? separator is `;`, not `,` (HTTP Cookie is not a generic list header)
