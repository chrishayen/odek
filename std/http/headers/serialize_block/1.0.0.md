# std.http.headers.serialize_block

Builds a full HTTP header block (terminated by a blank CRLF) from a name/value map.

## Signature

(headers: map[string, string]) -> result[string, string]

## Behavior

Emits every entry via `serialize_line` in iteration order, then appends a terminating CRLF. An empty map emits just "\r\n".

## Tests

+ emits each entry as a serialize_line output, in iteration order
+ emits a final blank CRLF after the last header
+ returns just "\r\n" for an empty map
- returns error when any entry fails serialize_line

## Assumptions

? iteration order is map insertion order
? blank line is exactly one CRLF after the last header's CRLF
