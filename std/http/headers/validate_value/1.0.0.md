# std.http.headers.validate_value

Returns true when a string is a valid RFC 7230 field-value (visible ASCII plus SP/HT, no CR/LF except in allowed obs-fold).

## Signature

(value: string) -> bool

## Behavior

Returns true iff every byte is either SP (0x20), HT (0x09), or in the visible-ASCII range 0x21-0x7E, or is a non-control byte above 0x7F (obs-text). Rejects CR, LF, and other control bytes.

## Tests

+ returns true for a simple ASCII value
+ returns true for values containing spaces and tabs
+ returns true for a value containing 0x80+ bytes (obs-text)
- returns false when the value contains CR or LF
- returns false when the value contains any control byte below 0x20 except HT
- returns false for a NUL byte

## Assumptions

? obs-fold (CRLF followed by whitespace) is rejected; modern HTTP bans it
