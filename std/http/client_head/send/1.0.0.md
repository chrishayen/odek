# std.http.client_head.send

Issues an HTTP HEAD request and returns the response metadata (status + headers, no body).

## Signature

(url: string, headers: map[string, string]) -> result[http_response, string]

## Behavior

Opens a connection, serializes a HEAD request, reads the response status and headers. Per RFC 7231, the server does not send a body for HEAD; the returned `http_response` has empty body bytes even if Content-Length indicates otherwise.

## Tests

+ returns the server's status and headers
+ returns an empty body regardless of Content-Length
+ accepts the same headers as client_get
- returns error when the URL is not http/https
- returns error when the server is unreachable

## Assumptions

? server must comply with HEAD and not send a body
