# std.http.client_head.parse_response_head

Reads and parses only the status line and headers of a HEAD response, then returns without reading a body.

## Signature

(conn: tcp_conn) -> result[http_response, string]

## Behavior

Reads the status line via the shared `parse_status_line`, then the header block via `parse_response_headers`. Returns an `http_response` with an empty body.

## Tests

+ returns a response with the correct status
+ returns the response headers
+ returns an empty body regardless of any Content-Length
- returns error when the status line fails to parse

## Assumptions

? connection remains positioned for keep-alive; no body bytes are consumed
