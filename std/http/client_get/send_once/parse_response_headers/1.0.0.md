# std.http.client_get.send_once.parse_response_headers

Reads the header block of an HTTP response off a connection and returns it as a name/value map.

## Signature

(conn: tcp_conn) -> result[map[string, string], string]

## Behavior

Reads bytes from the connection until the CRLF CRLF header terminator is seen, then delegates to the shared header block parser. Returns the same map shape as the server-side equivalent: lowercased names, trimmed values, duplicates joined with ", ".

## Tests

+ returns every response header in a map keyed by the lowercased name
+ stops reading at the CRLF CRLF terminator
+ advances the connection cursor to the first body byte
- returns error when the header block exceeds an implementation limit before terminator
- returns error when a header line fails to parse

## Assumptions

? reuses parse_header_line / parse_header_block from the server side for consistency
? header block size limit is implementation-defined
