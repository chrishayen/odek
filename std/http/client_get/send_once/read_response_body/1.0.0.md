# std.http.client_get.send_once.read_response_body

Reads the body of an HTTP response off a connection per its Content-Length or Transfer-Encoding header.

## Signature

(conn: tcp_conn, headers: map[string, string]) -> result[bytes, string]

## Behavior

Classifies the body framing via `determine_body_length`, then dispatches to `read_body_fixed` for Content-Length bodies, `read_body_chunked` for chunked bodies, or reads until connection close when neither header is present (HTTP/1.0 style). Returns the complete body.

## Tests

+ returns the exact body bytes for a Content-Length response
+ returns the reassembled body for a chunked response
+ reads until EOF when neither Content-Length nor Transfer-Encoding is present
- returns error when determine_body_length rejects a conflict
- returns error when a fixed-length read closes short of content-length

## Assumptions

? HTTP/1.0 style read-to-EOF requires the server to close after the body
? maximum body size is enforced by the caller
