# std.http.client_get.follow_redirects.is_redirect_status

Returns true when an HTTP status code represents a redirect that GET should follow.

## Signature

(code: i32) -> bool

## Behavior

Returns true for 301, 302, 303, 307, and 308. Returns false for all other codes, including 304 (Not Modified) and 300 (Multiple Choices), which do not mandate a followable Location.

## Tests

+ returns true for 301 and 302
+ returns true for 307 and 308
+ returns true for 303
- returns false for 304
- returns false for 300
- returns false for 200

## Assumptions

? classification is GET-method-specific; POST would differ on 307/308 preservation
? 300 and 304 are intentionally excluded
