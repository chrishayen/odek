# std.http.client_get.follow_redirects.resolve_location

Resolves a Location header value against the previous request URL to produce an absolute URL.

## Signature

(base: url, location: string) -> result[url, string]

## Behavior

Parses the `Location` value. If it is already an absolute URL (has a scheme), returns it as-is. If it starts with `//`, inherits the base scheme. If it starts with `/`, inherits the base scheme and authority. Otherwise treats it as path-relative to the base's path.

## Tests

+ returns the input as-is when it is already absolute
+ inherits the base scheme when the input starts with "//"
+ inherits the base scheme and authority when the input starts with "/"
+ resolves path-relative references against the base path
- returns error when the input is empty
- returns error when the input cannot be parsed as a URL or URL reference

## Assumptions

? resolution follows RFC 3986 section 5.2
? fragment of the base URL is dropped unless the input is fragment-only
