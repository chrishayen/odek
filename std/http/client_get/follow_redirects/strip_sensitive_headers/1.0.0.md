# std.http.client_get.follow_redirects.strip_sensitive_headers

Returns a copy of a headers map with authentication-related headers removed when the next URL is cross-origin.

## Signature

(headers: map[string, string], prev: url, next: url) -> map[string, string]

## Behavior

Compares the scheme+host+port of `prev` and `next`. When they differ, removes `authorization` and `cookie` from the returned map. When they match, returns the map unchanged. Comparison is case-insensitive on scheme and host.

## Tests

+ returns the input unchanged when prev and next share scheme+host+port
+ removes "authorization" on a cross-origin redirect
+ removes "cookie" on a cross-origin redirect
+ treats different ports on the same host as cross-origin
- returns a map with the same keys as input when the redirect stays on-origin

## Assumptions

? header-name matching is case-insensitive (names normalized to lowercase)
? port comparison uses the scheme default (80/443) when absent
