# std.http.client_get.follow_redirects

Follows an HTTP 3xx chain up to a bounded number of hops, reissuing GETs via a caller-supplied `send` function.

## Signature

(initial: http_response, send: func(string, map[string, string]) -> result[http_response, string], max_hops: i32) -> result[http_response, string]

## Behavior

If `initial` is not a 3xx response, returns it immediately. Otherwise resolves the `Location` header relative to the original URL, invokes `send` with the new URL, and repeats. Returns the first non-3xx response encountered. Strips `Authorization` and cookie headers on cross-origin redirects.

## Tests

+ returns the initial response unchanged when status is not 3xx
+ follows a single 301 redirect and returns the target response
+ follows up to max_hops redirects in sequence
+ resolves relative Location URLs against the previous request URL
+ strips Authorization header on a cross-origin redirect
- returns error when the redirect chain exceeds max_hops
- returns error when a 3xx response has no Location header
- returns error when a Location header cannot be parsed as a URL

## Assumptions

? max_hops is treated as at-most (0 means "do not follow anything")
? 307/308 preserve method; for GET this is identical to 301/302
