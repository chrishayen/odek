# std.http.multipart.format_part

Builds one multipart part — header block + CRLF CRLF + body — ready to be joined with boundaries.

## Signature

(headers: map[string, string], body: bytes) -> result[bytes, string]

## Behavior

Serializes the headers via `serialize_block` (which emits the trailing blank CRLF), then appends `body`. Does NOT emit the boundary marker lines; that's the caller's responsibility so they can sequence multiple parts.

## Tests

+ returns header block followed by body bytes
+ returns just the header block + CRLF when body is empty
- returns error when headers fail to serialize
- does not include boundary markers

## Assumptions

? caller inserts `--<boundary>\r\n` and trailing markers
