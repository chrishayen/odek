# std.http.cookies.parse_cookie_header

Parses a client-sent `Cookie` header value into a name→value map. Thin wrapper over the headers-namespace parser for discoverability.

## Signature

(value: string) -> result[map[string, string], string]

## Behavior

Delegates to `std.http.headers.specific.parse_cookie` unchanged. Exposed here so cookie-handling code can be discovered under `http.cookies` without learning the shared header namespace.

## Tests

+ parses "a=1; b=2" into the expected map
+ returns the same result as the underlying parse_cookie
- returns error on malformed input (same as parse_cookie)

## Assumptions

? this is a facade; the canonical implementation lives under http.headers.specific
