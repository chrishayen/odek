# std.http.cookies.format_cookie_header

Formats a map of name→value cookie pairs as a client-side `Cookie` header value.

## Signature

(cookies: map[string, string]) -> string

## Behavior

Joins each `<name>=<value>` pair with `"; "` in iteration order. Emits an empty string for an empty map.

## Tests

+ emits "a=1; b=2" for {"a": "1", "b": "2"}
+ emits "" for an empty map
+ preserves map iteration order
- does not emit attributes (those only apply to Set-Cookie)

## Assumptions

? cookie names and values are already validated by the caller
? no URL-encoding; callers encode if they need it
