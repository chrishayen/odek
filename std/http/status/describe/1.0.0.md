# std.http.status.describe

Returns the canonical reason-phrase for an HTTP status code, or "Unknown" when no phrase is registered.

## Signature

(status: i32) -> string

## Behavior

Looks up the IANA-registered reason phrase. Returns the phrase (e.g. "OK" for 200, "Not Found" for 404). Returns "Unknown" for codes not in the registry.

## Tests

+ returns "OK" for 200
+ returns "Not Found" for 404
+ returns "Internal Server Error" for 500
+ returns "Unknown" for an unregistered code (e.g. 999)
- returns no empty string (uses "Unknown" instead)

## Assumptions

? reason phrases are the IANA-registered values as of RFC 9110
? callers who need custom phrases provide their own
