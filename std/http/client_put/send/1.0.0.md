# std.http.client_put.send

Issues an HTTP PUT request and returns the response.

## Signature

(url: string, headers: map[string, string], body: bytes) -> result[http_response, string]

## Behavior

Same orchestration as `client_post.send` but uses the PUT method in the serialized request line. Sets Content-Length automatically. Does not follow redirects.

## Tests

+ issues a PUT with the supplied body
+ sets Content-Length automatically
+ returns the server's response verbatim
+ accepts an empty body (Content-Length: 0)
- returns error when the URL is not http or https
- returns error when the server is unreachable

## Assumptions

? PUT is idempotent; callers relying on that must ensure the server honors it
