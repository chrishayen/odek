# std.http.serve.handle_connection

Drives the per-connection HTTP/1.1 lifecycle: parse, dispatch to the handler, write, then loop while keep-alive holds.

## Signature

(conn: tcp_conn, handler: func(http_request) -> http_response) -> void

## Behavior

Repeatedly parses one request, invokes the handler to produce a response, and writes the response back. Continues until the client sends `Connection: close`, the request is HTTP/1.0 without `Connection: keep-alive`, a parse error occurs (answers 400 and closes), or the connection drops. Always closes the connection before returning.

## Tests

+ dispatches each parsed request to the handler on the connection
+ writes the handler's response bytes back to the client
+ honors HTTP/1.1 keep-alive across multiple requests on one connection
+ closes the connection after a "Connection: close" request/response pair
+ answers a 400 response and closes when parse_request returns error
- never returns without closing the connection

## Assumptions

? request-bodies larger than an implementation limit close the connection with 413
? handler exceptions translate to a 500 response
