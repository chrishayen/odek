# std.http.serve.parse_request.determine_body_length

Inspects HTTP request headers and classifies how the body is framed.

## Signature

(headers: map[string, string]) -> result[body_length, string]

## Behavior

Returns one of three variants: `chunked` when `transfer-encoding` is `chunked`, `fixed(n)` when `content-length` is present and parses as a non-negative integer, or `none` when neither header is present. Rejects any combination where both headers are present or where Content-Length is negative or non-numeric.

## Tests

+ returns `chunked` when transfer-encoding is "chunked"
+ returns `fixed(42)` when content-length is "42"
+ returns `none` when neither header is present
- returns error when both transfer-encoding and content-length are present
- returns error when content-length is negative
- returns error when content-length is not a decimal integer

## Assumptions

? transfer-encodings other than "chunked" (e.g. "gzip") are not accepted
? single-digit whitespace around Content-Length is trimmed before parsing
