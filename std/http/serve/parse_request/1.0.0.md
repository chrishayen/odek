# std.http.serve.parse_request

Reads and parses a single HTTP/1.1 request off a connection.

## Signature

(conn: tcp_conn) -> result[http_request, string]

## Behavior

Reads from the connection until the end of the request headers, parses the request line and header block, and reads the body per `Content-Length` or `Transfer-Encoding: chunked`. Returns a fully materialized `http_request`. Does not attempt multi-request pipelining — the connection cursor advances exactly one request.

## Tests

+ parses a GET request with headers and an empty body
+ parses a POST with a Content-Length body and returns the exact body bytes
+ parses a request with Transfer-Encoding: chunked and reassembles the body
+ preserves header names case-insensitively but keeps their values verbatim
+ advances the connection cursor to the byte after the request body
- returns error when the request line is malformed (caller responds 400)
- returns error when a header line exceeds the implementation's limit
- returns error when the connection closes mid-headers
- returns error when Content-Length and Transfer-Encoding conflict

## Assumptions

? protocol is HTTP/1.1 only
? header line-length and total-header-size limits are implementation-defined
