# std.http.serve.parse_request.read_chunk

Reads one chunked-encoding data chunk of a known size and consumes its trailing CRLF.

## Signature

(conn: tcp_conn, size: i64) -> result[bytes, string]

## Behavior

Reads exactly `size` bytes of chunk data from the connection, then consumes the two trailing bytes that must be CRLF. Returns only the data bytes; the terminator is discarded.

## Tests

+ returns exactly `size` bytes for the chunk data
+ consumes the CRLF following the chunk data
+ returns an empty buffer when size is 0 and the trailing CRLF is consumed
- returns error when the trailing bytes are not CRLF
- returns error when the connection closes mid-chunk

## Assumptions

? the size-0 terminating chunk is handled by read_body_chunked, not here
? CRLF is exactly "\r\n"; bare LF is rejected
