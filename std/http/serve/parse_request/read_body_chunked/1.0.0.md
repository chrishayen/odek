# std.http.serve.parse_request.read_body_chunked

Reads a complete chunked Transfer-Encoding body from a connection and returns it as a single buffer.

## Signature

(conn: tcp_conn) -> result[bytes, string]

## Behavior

Reads repeating pairs of (chunk-size line, chunk-data-plus-CRLF) and concatenates each chunk's data. Stops when a chunk-size of 0 is read and consumes the final CRLF that ends the trailer block.

## Tests

+ returns the concatenation of every chunk's data in order
+ returns an empty buffer when the first chunk has size 0
- returns error when a chunk-size line is not valid hex
- returns error when a data chunk is missing its trailing CRLF

## Assumptions

? trailer headers after the 0-size chunk are read and discarded
? maximum total body size is enforced by the caller, not this rune
