# std.http.serve.write_response

Serializes an HTTP/1.1 response and writes it to a connection.

## Signature

(conn: tcp_conn, response: http_response) -> result[void, string]

## Behavior

Writes the status line, headers, and body in order. Sets `Content-Length` automatically from the body unless the response already has a `Transfer-Encoding` header. Ensures CRLF line endings throughout. Does not close the connection; keep-alive continues until the caller decides otherwise.

## Tests

+ writes a well-formed HTTP/1.1 response for the given status and body
+ sets Content-Length automatically when absent
+ preserves caller-supplied headers verbatim
+ writes bytes in order: status line, headers, blank line, body
- returns error when the connection is closed mid-write
- returns error when the response status code is outside 100..599

## Assumptions

? line endings are CRLF
? the caller decides whether to close or keep the connection open
