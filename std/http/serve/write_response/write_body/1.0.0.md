# std.http.serve.write_response.write_body

Writes the body bytes of an HTTP response to a connection.

## Signature

(conn: tcp_conn, body: bytes) -> result[void, string]

## Behavior

Issues repeated writes against the connection until every byte of `body` has been sent. An empty body performs no writes. Does not add any framing — the caller has already set Content-Length or Transfer-Encoding via the header block.

## Tests

+ writes every byte of the body in order
+ performs no writes when the body is empty
+ retries on short writes until the full body is flushed
- returns error when the connection closes mid-write

## Assumptions

? no compression or encoding is applied here
? short writes are retried transparently
