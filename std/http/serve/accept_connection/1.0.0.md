# std.http.serve.accept_connection

Accepts one inbound TCP connection from an HTTP listening socket.

## Signature

(listener: tcp_listener) -> result[tcp_conn, string]

## Behavior

Blocks until a client completes the TCP handshake, then returns the new connection. The listener remains open and can be used for subsequent accepts. The returned connection inherits the listener's default socket options.

## Tests

+ returns a connected tcp_conn for each successful client handshake
+ blocks until a client connects when the listener has no pending connections
+ returns independent connections across multiple accepts
- returns error when the listener has been closed
- returns error when accept fails at the OS level (e.g. file-descriptor exhaustion)

## Assumptions

? inherits the listener's keep-alive, nodelay, and timeout defaults
? accept is blocking; non-blocking variants are a separate rune
