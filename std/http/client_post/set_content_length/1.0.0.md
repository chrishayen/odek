# std.http.client_post.set_content_length

Returns a copy of a request-headers map with `Content-Length` set to the body length, unless the caller has already set Transfer-Encoding.

## Signature

(headers: map[string, string], body: bytes) -> map[string, string]

## Behavior

When the input headers contain `transfer-encoding`, returns them unchanged. Otherwise returns a copy with `content-length` set to the decimal byte length of `body`, replacing any existing Content-Length.

## Tests

+ sets content-length equal to len(body) when transfer-encoding is absent
+ returns headers unchanged when transfer-encoding is present
+ overwrites a caller-supplied content-length with the computed value
+ sets content-length to "0" for an empty body
- does not mutate the input headers map

## Assumptions

? header name lookup is case-insensitive
? when both headers would be present, transfer-encoding wins
