# std.time.parse_rfc3339.compose_millis

Combines day count, time-of-day, fractional milliseconds, and UTC offset into a Unix-epoch millisecond count.

## Signature

(days: i64, tod: time_of_day, millis: i32, offset_minutes: i32) -> i64

## Behavior

Computes `days * 86_400_000 + tod.hour * 3_600_000 + tod.minute * 60_000 + tod.second * 1000 + millis`, then subtracts `offset_minutes * 60_000` to normalize to UTC.

## Tests

+ returns 0 for days=0, tod=(0,0,0), millis=0, offset=0
+ returns 86_400_000 for days=1, tod=(0,0,0), millis=0, offset=0
+ subtracts 3_600_000 for a +01:00 offset at the same local wall time
- returns a value that reflects offset subtraction, not addition

## Assumptions

? constants use the standard definition of 86,400,000 milliseconds per day
? inputs are already validated by upstream runes
