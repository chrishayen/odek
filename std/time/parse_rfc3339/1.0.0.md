# std.time.parse_rfc3339

Parses an RFC 3339 timestamp string into a Unix-epoch millisecond count.

## Signature

(s: string) -> result[i64, string]

## Behavior

Accepts RFC 3339 timestamps with a required date, required time, optional fractional seconds, and a required offset (`Z` or `±HH:MM`). Normalizes to UTC and returns milliseconds since the Unix epoch. Fractional seconds beyond millisecond precision are truncated.

## Tests

+ parses "2025-01-15T12:00:00Z" to the correct epoch millisecond count
+ parses "2025-01-15T12:00:00+02:00" and normalizes to UTC
+ parses "2025-01-15T12:00:00.123Z" preserving millisecond precision
+ accepts "z" and "Z" as equivalent UTC markers
- returns error when the string is missing an offset
- returns error when the month or day is out of range
- returns error on an empty string
- returns error when the separator between date and time is neither "T" nor " "

## Assumptions

? fractional precision beyond milliseconds is truncated, not rounded
? leap seconds are not represented (second=60 is rejected)
