# std.time.parse_rfc3339.parse_date

Parses a `YYYY-MM-DD` date token into year/month/day integers.

## Signature

(token: string) -> result[date, string]

## Behavior

Accepts a 10-character token. Requires four-digit year, `-`, two-digit month (01-12), `-`, two-digit day (01-31, with month-aware day limits and leap-year-aware February). Returns a `date` struct on success.

## Tests

+ returns year 2025, month 1, day 15 for "2025-01-15"
+ accepts year 0000 and years up to 9999
+ allows day 29 in February of leap years
- returns error for a month outside 01-12
- returns error for a day outside the month's valid range
- returns error for day 29 in February of a non-leap year
- returns error when the separators are not "-"

## Assumptions

? Gregorian calendar; leap years per the 4/100/400 rule
? year is always four digits; no BCE representation
