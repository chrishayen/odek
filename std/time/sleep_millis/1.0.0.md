# std.time.sleep_millis

Blocks the calling thread for at least the given number of milliseconds.

## Signature

(duration: i64) -> void

## Behavior

Pauses execution of the calling thread and yields to the scheduler for at least `duration` milliseconds. May sleep slightly longer due to scheduler granularity. A non-positive duration returns immediately. Interruptions by signals do not shorten the wait — the function resumes sleeping for the remaining time.

## Tests

+ blocks for at least `duration` milliseconds before returning
+ returns immediately for a duration of 0
+ returns immediately for a negative duration
+ resumes sleeping after a signal interruption until the full duration elapses
- returns before `duration` milliseconds under no non-error circumstance

## Assumptions

? resolution is bounded below by OS scheduler granularity (often ~1-10ms)
? cancellation is not supported; once invoked, the call always completes the requested delay
