# std.crypto.pbkdf2

Derives a key of the requested length from a password using PBKDF2-HMAC-SHA-256.

## Signature

(password: bytes, salt: bytes, iterations: i32, output_length: i32) -> result[bytes, string]

## Behavior

Applies PBKDF2 per RFC 2898 with HMAC-SHA-256 as the PRF. Uses the given iteration count. Returns `output_length` bytes of derived key.

## Tests

+ returns exactly output_length bytes
+ produces deterministic output for the same inputs
+ different salts yield different outputs
+ different iteration counts yield different outputs
- returns error when iterations < 1
- returns error when output_length is not positive
- returns error when salt is empty

## Assumptions

? PRF is HMAC-SHA-256; a separate rune could parameterize the PRF
? modern recommendation: iterations >= 600_000
