# std.crypto.hkdf

Derives a key from input keying material using HKDF (HMAC-SHA-256).

## Signature

(ikm: bytes, salt: bytes, info: bytes, output_length: i32) -> result[bytes, string]

## Behavior

Applies HKDF-Extract-then-Expand per RFC 5869 with SHA-256. Output up to 255 * 32 = 8160 bytes. `salt` may be empty (treated as zero-salt). `info` may be empty.

## Tests

+ returns exactly output_length bytes
+ produces deterministic output for the same inputs
+ empty salt is valid (treated as zero)
+ different info values yield different outputs
- returns error when output_length > 8160
- returns error when output_length is not positive

## Assumptions

? hash is SHA-256; other hashes are separate runes
