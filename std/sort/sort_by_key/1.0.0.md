# std.sort.sort_by_key

Returns a new list sorted by the value returned from a key-extraction function applied to each element.

## Signature

(xs: list[T], key: func(T) -> K, less: func(K, K) -> bool) -> list[T]

## Behavior

For each element, extracts a key via `key`. Sorts elements so that their keys are ascending under `less`. Caches keys so `key` is called exactly once per element.

## Tests

+ sorts a list of records ascending by an extracted field
+ returns an empty list for an empty input
+ calls key exactly once per element
+ handles equal keys — their relative order follows the implementation's stability
- does not mutate the input

## Assumptions

? key is pure (no side effects) so caching is safe
? stability is not guaranteed; composes with sort_slice_stable if needed
