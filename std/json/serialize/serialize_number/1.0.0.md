# std.json.serialize.serialize_number

Formats a JSON number (i64 or f64) into its canonical JSON literal form.

## Signature

(n: json_number) -> string

## Behavior

For an i64, emits the decimal representation with no leading zeros and an optional `-` sign. For an f64, emits the shortest round-trippable decimal form; uses exponent notation for very large or small values.

## Tests

+ emits "0" for integer 0
+ emits "-123" for integer -123
+ emits "3.14" for float 3.14
+ emits exponent notation for very large floats (e.g. 1e20)
- emits no leading zeros (e.g. "007" is not produced)
- emits nothing for NaN (NaN input is rejected by an upstream validator)

## Assumptions

? NaN and infinity are not representable in JSON; the caller must filter them
? float formatting is round-trippable (input recovers via parse_number)
