# std.json.parse.parse_number

Parses a JSON number literal and returns its value (as i64 or f64) and the position after it.

## Signature

(input: string, pos: i64) -> result[number_result, string]

## Behavior

Scans per RFC 8259 grammar: optional `-`, integer part (`0` or `1-9` followed by digits), optional fractional part (`.` + digits), optional exponent (`e|E` + optional sign + digits). Emits `i64` when the literal has neither fraction nor exponent and fits in i64; emits `f64` otherwise.

## Tests

+ parses "0" as integer 0
+ parses "-123" as integer -123
+ parses "3.14" as float 3.14
+ parses "1e2" as float 100.0
- returns error on a leading zero followed by more digits ("01")
- returns error on a bare "." without digits on either side
- returns error on an empty exponent ("1e")

## Assumptions

? integer overflow of i64 falls back to f64 representation
? NaN and infinity are not representable in JSON and are rejected
