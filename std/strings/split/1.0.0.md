# std.strings.split

Splits the input string into a list of substrings separated by the given separator.

## Signature

(s: string, sep: string) -> list[string]

## Behavior

Scans the input for non-overlapping occurrences of the separator and returns the substrings between them. Adjacent separators produce empty-string segments. The separator itself is not included in any returned segment. When the separator does not appear in the input, a single-element list containing the original string is returned.

## Tests

+ returns ["a", "b", "c"] for input ("a,b,c", ",")
+ returns ["hello"] when the separator is absent
+ returns ["", "a", "b"] when the input starts with the separator
+ returns ["a", "b", ""] when the input ends with the separator
+ returns ["", ""] for input ("", "") — single empty segment pair when both are empty
+ returns per-character segments when sep is the empty string and s is non-empty
- returns [""] for an empty input with a non-empty separator

## Assumptions

? separator matching is byte-exact; no regex interpretation
? an empty separator causes code-point-level segmentation
