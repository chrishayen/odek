# std.regex.compile.tokenize

Splits a regex pattern string into a flat list of lexical tokens (literal, metachar, group-open, group-close, etc.).

## Signature

(pattern: string) -> result[list[regex_token], string]

## Behavior

Walks the pattern left-to-right. Emits one token per significant unit: a single literal character, a metacharacter (`.`, `^`, `$`, `|`, `?`, `*`, `+`, `(`, `)`, `[`, `]`, `{`, `}`), or an escape sequence. Does not build structure — just a token stream.

## Tests

+ returns [lit(a), lit(b), lit(c)] for pattern "abc"
+ returns tokens for each metacharacter in a pattern
+ treats `\.` as a literal `.`
- returns error on a trailing backslash with no escape character
- returns error on a token that the lexer does not recognize

## Assumptions

? token stream preserves source position for later diagnostics
? unicode characters are treated as single tokens
