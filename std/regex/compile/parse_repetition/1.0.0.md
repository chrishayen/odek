# std.regex.compile.parse_repetition

Recognizes a repetition suffix (`?`, `*`, `+`, or `{n,m}`) at a token position and returns min/max counts.

## Signature

(tokens: list[regex_token], pos: i64) -> result[repetition_result, string]

## Behavior

Inspects the token at `pos`. Returns `(0, 1)` for `?`, `(0, ∞)` for `*`, `(1, ∞)` for `+`. For `{n,m}` returns `(n, m)`. Returns none when the token is not a repetition. Advances pos past the suffix.

## Tests

+ returns (0, ∞) for `*`
+ returns (1, ∞) for `+`
+ returns (0, 1) for `?`
+ returns (2, 5) for `{2,5}`
+ returns (3, 3) for `{3}`
- returns error on `{2,1}` where min > max

## Assumptions

? `{n,}` means `(n, ∞)`; `{,m}` means `(0, m)`
? greediness is assumed greedy; lazy `?` suffix is a separate rune
