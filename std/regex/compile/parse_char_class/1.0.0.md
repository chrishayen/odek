# std.regex.compile.parse_char_class

Parses a character class `[...]` from a token stream and returns its set of matchable bytes.

## Signature

(tokens: list[regex_token], pos: i64) -> result[char_class_result, string]

## Behavior

Expects `[` at `pos`. Reads tokens, accumulating individual characters and `a-z` ranges into a set. Handles leading `^` as negation. Consumes the closing `]`.

## Tests

+ parses "[abc]" as the set {a, b, c}
+ parses "[a-z]" as the set of every lowercase letter
+ parses "[^0-9]" as every character except digits
- returns error on an unbalanced bracket
- returns error on an inverted range like "[z-a]"

## Assumptions

? ranges are inclusive on both ends
? `]` immediately after `[` or `[^` is treated as a literal `]` in the class
