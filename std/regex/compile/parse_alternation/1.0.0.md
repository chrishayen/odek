# std.regex.compile.parse_alternation

Parses an `|`-separated list of sub-expressions and returns them as a flat list of alternatives.

## Signature

(tokens: list[regex_token], pos: i64) -> result[alternation_result, string]

## Behavior

Reads sub-expressions until the next `|` is absent or a closing `)` is encountered. Returns the list of alternatives. A single expression with no `|` is returned as a one-element list.

## Tests

+ returns [a, b] for pattern "a|b"
+ returns [a, b, c] for pattern "a|b|c"
+ returns [a] for pattern "a" (single alternative, no bars)
- returns error when an alternative is empty (e.g. "a||b")

## Assumptions

? precedence: concatenation is tighter than alternation (standard)
