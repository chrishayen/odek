# std.regex.split

Splits the input string on every non-overlapping regex match and returns the resulting segments.

## Signature

(re: regex, s: string) -> list[string]

## Behavior

Scans `s` for non-overlapping matches of `re` and returns the substrings between them. The matched text is not included in any segment. When the regex does not match, the result is a single-element list containing the original string.

## Tests

+ returns ["a", "b", "c"] for pattern "," on input "a,b,c"
+ returns ["a", "b", "c"] for pattern "\\s+" on input "a  b\tc"
+ returns [""] when the regex matches the whole input
+ returns [s] when the regex does not match
- produces no segments of regex-matched text; only the gaps between matches

## Assumptions

? adjacent matches produce empty segments
? empty-match semantics advance the cursor by one code point
