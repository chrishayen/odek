# std.concurrency.channel.close

Closes a channel so that no further values can be sent, and receivers drain until empty.

## Signature

(ch: channel[T]) -> result[void, string]

## Behavior

Marks the channel as closed. Existing buffered values remain available to receivers; subsequent receives after draining return `none`. Subsequent send calls fail with an error. Closing an already-closed channel is an error.

## Tests

+ subsequent receives drain the buffer, then return none
+ subsequent sends return error
+ unblocks any currently blocked receivers with none after the buffer drains
- returns error when the channel is already closed
- returns error when called from a non-owner when the implementation restricts ownership

## Assumptions

? close is idempotent only in implementations that opt in; the default is to error on double-close
? multiple senders without coordination may race on close; caller serializes
