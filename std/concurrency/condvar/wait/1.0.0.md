# std.concurrency.condvar.wait

Atomically releases a mutex, blocks until the condition variable is signaled, then re-acquires the mutex before returning.

## Signature

(cv: condvar, m: mutex) -> void

## Behavior

Must be called with `m` held. Atomically drops `m` and suspends. When signaled, re-acquires `m` (possibly blocking again) before returning. Callers should re-check their predicate in a loop because of spurious wakeups.

## Tests

+ releases the mutex before suspending
+ re-acquires the mutex before returning
+ is unblocked by signal or broadcast
+ may wake spuriously; callers must re-check the predicate
- produces undefined behavior when m is not held on entry

## Assumptions

? spurious wakeups are possible; callers wrap in a while-loop around the predicate
