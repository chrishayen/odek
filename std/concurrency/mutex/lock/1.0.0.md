# std.concurrency.mutex.lock

Acquires a mutex, blocking until it is available.

## Signature

(m: mutex) -> void

## Behavior

Blocks the calling thread until the mutex is uncontended, then acquires exclusive ownership. Ownership is thread-local: only the thread that acquired the lock may release it. Re-entry on the same thread is not permitted (the call deadlocks) unless the mutex is explicitly recursive.

## Tests

+ acquires the lock immediately when uncontended
+ blocks until another thread releases the lock when contended
+ eventually acquires when a holding thread releases
+ preserves FIFO-ish fairness under contention (implementation-defined)
- deadlocks when the same thread attempts to re-lock a non-recursive mutex

## Assumptions

? mutex is non-recursive by default; use a separate recursive mutex if re-entry is needed
? fairness is best-effort; no strict FIFO guarantee
