# std.concurrency.mutex.unlock

Releases a mutex previously acquired by the calling thread.

## Signature

(m: mutex) -> void

## Behavior

Releases exclusive ownership of the mutex and allows another waiting thread to proceed. Only the thread that acquired the lock may call unlock; calling from another thread, or on a mutex that was never locked, is undefined behavior.

## Tests

+ releases the lock so a waiting thread can acquire
+ allows the same thread to re-lock after release
+ does not wake unblocked threads out of order (implementation-defined fairness)
- produces undefined behavior when called by a non-owner thread
- produces undefined behavior when the mutex was never locked

## Assumptions

? ownership rule is enforced only in debug builds; release builds may corrupt state if violated
