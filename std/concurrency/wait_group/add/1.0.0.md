# std.concurrency.wait_group.add

Increments a wait group's counter by `n`, marking additional tasks pending.

## Signature

(wg: wait_group, n: i32) -> void

## Behavior

Adds `n` to the counter. Must be called before the corresponding `done` calls, typically before spawning the worker. Negative `n` is allowed but must not drive the counter below 0.

## Tests

+ increments the counter by n
+ accepts zero n as a no-op
+ accepts negative n to cancel pending adds (counter stays >= 0)
- produces undefined behavior when called concurrently with wait

## Assumptions

? add is not safe to race with a pending wait; add before spawn
