# std.concurrency.atomic.add_i64

Atomically adds a delta to an atomic i64 cell and returns the previous value.

## Signature

(cell: atomic_i64, delta: i64) -> i64

## Behavior

Performs an atomic fetch-and-add: reads the current value, computes value + delta, stores it back, and returns the pre-add value. The operation is linearizable with respect to other atomic operations on the same cell. Uses sequentially consistent ordering unless the host provides a weaker-order variant.

## Tests

+ returns the pre-add value
+ stores value + delta in the cell
+ is linearizable: under concurrent callers, the sum of all deltas equals (final - initial)
+ handles two's-complement wrap-around for overflow
- produces no torn reads or writes under concurrent access

## Assumptions

? memory order is sequentially consistent
? wrap-around on i64 overflow is defined (two's complement, no trap)
