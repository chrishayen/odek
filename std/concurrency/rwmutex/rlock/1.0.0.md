# std.concurrency.rwmutex.rlock

Acquires a read lock on the mutex, blocking if a writer is active.

## Signature

(m: rwmutex) -> void

## Behavior

Multiple threads can hold the read lock simultaneously. Blocks if a writer currently holds the write lock or is waiting for one (to prevent writer starvation).

## Tests

+ allows multiple concurrent readers to hold simultaneously
+ blocks while a writer holds the lock
+ blocks while a writer is waiting (to prevent writer starvation)
- does not deadlock when many readers acquire concurrently

## Assumptions

? writer-preference policy prevents reader starvation of writers
