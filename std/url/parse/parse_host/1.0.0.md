# std.url.parse.parse_host

Validates and returns the host portion of a URL authority.

## Signature

(token: string) -> result[string, string]

## Behavior

Accepts the substring after optional userinfo and before optional port. Recognizes three forms: IP-literal `[v6-or-future]`, IPv4 dotted-decimal, or reg-name (DNS-style). Returns the host lowercased when it is a reg-name; leaves IP literals unchanged.

## Tests

+ returns "example.com" for input "EXAMPLE.COM" (lowercased reg-name)
+ returns "1.2.3.4" for IPv4 input "1.2.3.4"
+ returns "[::1]" for IPv6 input "[::1]" with brackets preserved
- returns error on an empty input
- returns error when a reg-name contains a character outside the unreserved/sub-delims/pct-encoded set

## Assumptions

? reg-names are lowercased; IP literals are preserved verbatim
? IDN hosts are assumed already Punycode-encoded
