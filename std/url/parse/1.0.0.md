# std.url.parse

Parses an absolute URL string into its structured components.

## Signature

(raw: string) -> result[url, string]

## Behavior

Decomposes the input into scheme, optional user-info, host, optional port, path, optional query, and optional fragment per RFC 3986. IDN hosts are preserved in Punycode form. Percent-encoded octets in the path, query, and fragment are left encoded; the caller decodes as needed.

## Tests

+ parses "https://example.com/a/b" into scheme="https", host="example.com", path="/a/b"
+ parses "https://u:p@example.com:8080/?q=1#frag" into all components
+ parses an IPv6 host like "https://[::1]:80/" with brackets preserved in host
+ parses a path-only URL when given a scheme-relative reference (e.g. "//example.com/x")
- returns error on a missing scheme for an absolute URL
- returns error on a malformed percent-encoding like "%2"
- returns error on a port that is not an integer

## Assumptions

? only absolute URLs are accepted; use a separate function for relative references
? hosts are stored lowercased
