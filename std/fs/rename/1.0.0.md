# std.fs.rename

Moves or renames a filesystem entry from one path to another.

## Signature

(src: string, dst: string) -> result[void, string]

## Behavior

Renames the entry at `src` to `dst`. On the same filesystem the operation is atomic: either the new name is created and the old one is removed, or neither happens. Cross-filesystem moves may fail with an error (a separate copy+delete is required).

## Tests

+ renames a file within the same directory
+ moves a file to a different directory on the same filesystem
+ replaces an existing destination file atomically
+ renames a directory to a new name
- returns error when src does not exist
- returns error when the destination's parent directory does not exist
- returns error when moving across filesystems (EXDEV)

## Assumptions

? atomic only when src and dst are on the same filesystem
? overwrite semantics follow the host OS (POSIX replaces, Windows may require delete-first)
