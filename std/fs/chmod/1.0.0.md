# std.fs.chmod

Changes a file's Unix-style permission bits.

## Signature

(path: string, mode: u32) -> result[void, string]

## Behavior

Applies the low 12 bits of `mode` to the file's permission bits. Upper bits are ignored. No-op on platforms without Unix-like permissions (e.g. Windows behavior is to set read-only based on writable-by-owner).

## Tests

+ sets the permission bits to the given mode
+ accepts standard mode values like 0644, 0755
- returns error when the path does not exist
- returns error when the caller is not the owner and lacks privilege

## Assumptions

? ignores bits above the low 12; sticky/setuid/setgid require the low 12 for representation
