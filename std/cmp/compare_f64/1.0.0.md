# std.cmp.compare_f64

Compares two f64 values under IEEE-754 total-order rules, or errors when either is NaN.

## Signature

(a: f64, b: f64) -> result[i32, string]

## Behavior

Returns -1 when `a < b`, 0 when `a == b`, 1 when `a > b`. Treats -0.0 as equal to +0.0. Returns an error when either input is NaN (NaN has no defined ordering).

## Tests

+ returns -1 for (1.0, 2.0)
+ returns 0 for (1.0, 1.0)
+ returns 0 for (-0.0, 0.0)
+ returns 1 for (2.0, 1.0)
- returns error when a is NaN
- returns error when b is NaN

## Assumptions

? -0.0 equals 0.0 for ordering purposes
? callers who want NaN-tolerant ordering use total_order_f64 (separate rune)
