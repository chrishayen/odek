# std.uuid.new_v7

Generates a time-ordered version-7 UUID (Unix-time-ms + random) in canonical string form.

## Signature

() -> string

## Behavior

Per draft-ietf-uuidrev-rfc4122bis: the first 48 bits are the current millisecond timestamp, followed by version (0111), 12 random bits, variant (10xx), and 62 random bits. Two v7 UUIDs generated in the same millisecond are ordered by the random suffix — collision-unlikely.

## Tests

+ returns a string of length 36
+ the first 48 bits encode the current millisecond timestamp
+ the version nibble is "7"
+ successive calls produce lexicographically increasing UUIDs (within timestamp resolution)
- emits no uppercase hex digits

## Assumptions

? timestamp is Unix milliseconds
? entropy is from std.crypto.random_bytes
