# std.process.run.resolve_executable

Resolves a command name to an absolute executable path, searching the host `PATH` when the command is not already absolute.

## Signature

(cmd: string) -> result[string, string]

## Behavior

When `cmd` is an absolute path, verifies it exists and is executable, then returns it unchanged. Otherwise reads `PATH` from the environment, searches each directory left-to-right for an executable file named `cmd`, and returns the first match as an absolute path.

## Tests

+ returns the input unchanged when it is an absolute path and executable
+ returns the first PATH directory match for a bare command name
+ searches PATH directories left-to-right (earlier directories win)
+ considers only files that are both present and executable by the process
- returns error when the command is not found in any PATH directory
- returns error when an absolute path does not exist
- returns error when an absolute path exists but is not executable

## Assumptions

? PATH separator is ":" on POSIX and ";" on Windows
? on Windows, extensions in PATHEXT are tried for bare names
