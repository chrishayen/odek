# std.process.run.capture

Drains a running child's stdout and stderr pipes to completion, returning both byte buffers.

## Signature

(handle: process_handle) -> result[captured_output, string]

## Behavior

Reads concurrently from the handle's stdout and stderr pipes until each reaches EOF. Accumulates each stream into its own byte buffer in arrival order. Does not wait for the child to exit — callers typically pair this with a wait rune. Does not interleave the two streams into one buffer.

## Tests

+ returns every byte emitted on stdout in arrival order
+ returns every byte emitted on stderr in arrival order
+ does not interleave stdout bytes into the stderr buffer or vice versa
+ returns empty buffers when the child produced no output
+ drains both pipes concurrently to avoid deadlock when one fills
- returns error when a pipe read fails before EOF
- returns error when the handle's pipes were already consumed

## Assumptions

? both buffers are held in memory; streaming variants are a separate rune
? the child may still be running when capture returns (pipes closed but exit not reaped)
