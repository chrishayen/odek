# std.process.run.wait

Blocks until the child referenced by the handle exits, then returns its exit code.

## Signature

(handle: process_handle) -> result[i32, string]

## Behavior

Waits for the child process to terminate and reaps it. Returns the non-negative exit code for a normal exit. When the child is terminated by a signal, returns `128 + signal_number` to mirror shell conventions. Once called successfully, the handle is consumed and subsequent waits return error.

## Tests

+ returns 0 for a child that exits with success
+ returns the non-zero exit code for a child that exits with a status
+ returns 128 + signal for a child terminated by a signal
+ blocks until the child actually exits
+ reaps the child, preventing a zombie
- returns error when the handle has already been waited on
- returns error when the OS loses track of the PID (e.g. reparented and reaped elsewhere)

## Assumptions

? the handle can only be waited on once
? signal-termination encoding follows shell convention (128 + N)
