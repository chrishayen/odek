# std.process.run.spawn

Spawns a child process from an absolute executable path with the given arguments, returning a handle whose stdout and stderr are pipes.

## Signature

(path: string, args: list[string]) -> result[process_handle, string]

## Behavior

Creates two pipes (for stdout and stderr), forks, and execs `path` with `args` as `argv[1..]` (the child's `argv[0]` is `path`). Returns a `process_handle` carrying the child's PID and both read-ends of the pipes. Does not invoke a shell; arguments are passed verbatim.

## Tests

+ returns a handle whose pid is positive and distinct from the parent
+ passes args verbatim as argv[1..]; sets argv[0] to path
+ leaves stdin inherited from the parent unless redirected separately
+ returns a handle with readable stdout and stderr pipe endpoints
- returns error when path is not executable at spawn time
- returns error when the OS process table is full
- returns error when arguments contain NUL bytes (not permitted in exec)

## Assumptions

? the child inherits the parent's working directory and environment unless extended
? pipe buffer size defaults to the OS default (typically 64KB)
