# std.compress.gzip_compress.write_trailer

Emits the 8-byte gzip trailer: CRC-32 and ISIZE, both little-endian.

## Signature

(crc32: u32, isize: u32) -> bytes

## Behavior

Returns 8 bytes: `crc32` as four little-endian bytes followed by `isize` as four little-endian bytes. `isize` is the uncompressed data length modulo 2^32.

## Tests

+ returns exactly 8 bytes
+ places CRC-32 in the first 4 bytes, little-endian
+ places ISIZE in the last 4 bytes, little-endian
- emits nothing in big-endian order

## Assumptions

? ISIZE is the caller's responsibility to compute modulo 2^32 before passing in
