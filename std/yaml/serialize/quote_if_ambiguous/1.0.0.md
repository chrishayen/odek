# std.yaml.serialize.quote_if_ambiguous

Decides whether a string needs to be quoted in YAML output to avoid being reinterpreted as another scalar type.

## Signature

(s: string) -> bool

## Behavior

Returns true when the string would be reparsed as null, bool, or number under the YAML 1.2 core schema, or when it contains characters with special block-meaning (leading/trailing whitespace, `:`, `#`, `-`, `?`, `[`, `]`, `{`, `}`, `,`, `&`, `*`, `!`, `|`, `>`, `'`, `"`, `%`, `@`, backtick). Returns false for safe plain scalars.

## Tests

+ returns true for "true", "false", "null", and "~"
+ returns true for numeric-looking strings like "42" and "3.14"
+ returns true for strings containing ": " or leading/trailing whitespace
+ returns false for simple alphabetic strings like "hello"
- returns false for strings that are unambiguous under the core schema

## Assumptions

? the test is conservative; some safe strings may still be quoted
