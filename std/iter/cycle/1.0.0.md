# std.iter.cycle

Returns an infinite iterator that repeats the elements of the input forever.

## Signature

(xs: iter[T]) -> iter[T]

## Behavior

Caches the full contents of the input iterator (consumes it first), then yields them in order, restarting from the beginning once the last element has been yielded. Never terminates unless the input is empty.

## Tests

+ yields the input's elements in order on the first pass
+ yields the input's elements again on subsequent passes
+ yields an empty iterator when the input is empty
- never yields a non-input element

## Assumptions

? the input must be collectible in memory; infinite-input cycle is undefined
